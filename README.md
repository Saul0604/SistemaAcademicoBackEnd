# 📚 Sistema Académico Simple

## 📌 Descripción General

**Sistema Académico Simple** es una aplicación web diseñada para la administración básica de una escuela. El sistema permite:

- ✅ Registro y gestión de alumnos
- 📚 Control y administración de materias
- 📊 Gestión y registro de calificaciones

El objetivo del proyecto es facilitar el control académico mediante una plataforma digital accesible y organizada.

## 🛠 Tecnologías Utilizadas

### Frontend
- ⚛️ React

### Backend
- 🟢 Node.js
- 🚀 Express

### Control de Versiones
- 🐙 GitHub

## 📋 Reglas de Trabajo

### 📢 Comunicación
- El canal oficial de comunicación será **WhatsApp**.
- Todas las decisiones importantes se tomarán y confirmarán por este medio.
- Se realizarán reuniones diarias para seguimiento del avance.

### 📋 Asignación de Tareas
- Se trabajará bajo la metodología **Scrum**.
- Se organizarán entregas mediante sprints semanales.
- Las tareas serán asignadas y supervisadas por el Scrum Master.

## 🔄 Reglas para Cambios

### 🔀 Control de Versiones
El proyecto se gestionará mediante GitHub. Se trabajará utilizando ramas:
- `main`
- `develop`
- `feature/nombre-feature`

### 🔎 Pull Requests
- Todo cambio debe realizarse mediante una rama.
- Se debe crear un Pull Request para integrar cambios.
- El Pull Request requiere mínimo **1 revisión**.
- El Scrum Master será el responsable de aprobar los cambios.
- Antes de hacer merge, el código debe ser probado para asegurar su correcto funcionamiento.

### 🧼 Estándar de Código
Aunque actualmente no se ha definido un estándar formal, el equipo se compromete a:
- Mantener código limpio y legible.
- Usar nombres descriptivos para variables y funciones.
- Mantener una estructura organizada del proyecto.

## ⚠️ Protocolo de Incumplimiento

En caso de que un integrante no cumpla con sus responsabilidades:
- Se realizarán hasta **3 llamadas de atención**.
- Cada incumplimiento será documentado.

Si después de las advertencias no hay cumplimiento, el integrante podrá ser removido del equipo.

Los casos se reportarán al docente en las entregas semanales.

El equipo podrá reasignar tareas para evitar retrasos en el proyecto.
